Critical constraint: anything in the backend that calls LocalLLMClient.quick_command() must acquire the mlx.lock flock first (committed in e0d27b2). But base footprint (mempalace ChromaDB 1–2 GB + pykeen PyTorch 300 MB + OS + Chrome + platform + Claude Code) is already 8–12 GB. Add Gemma4-26B load (~15 GB) and you're at 23–27 GB. Overlap & improvement candidates

Consolidation

  • morning-briefing × weekly-report × last-cron-result.json — three reports, same query sources. Share a reporting_core.py; each picks a time window.
  • email-watcher 14-step subprocess chain — 6–10 subprocess.run calls per bundle, each cold-imports Python. Collapse to in-process functions → 4–8× faster, far less fork storm at boot.
  • think × email-watcher split-brain queue — think writes queue.json, email-watcher writes pending_todos.json + pending_decisions.json. Platform has to merge. Unify to one queue (SQLite table or single JSON).
  • backup × litestream — litestream already gives 30d incremental replication. Weekly .backup is redundant unless you want a fully-independent snapshot.
  • wiki_improver runs outside cron — it's advertised as a continuous-improvement loop but needs a human to start it. Either wrap in a plist with StartInterval=21600 (6h) gated by an idle-check + flock, or delete and fold its passes into master-cron phase 11.

Risks the flock doesn't cover

  • mempalace + pykeen-train co-residency — both hold RAM continuously; neither takes the flock. Worst-case memory = base (mempalace 1.5 GB + pykeen 0.3 GB + platform 0.5 GB) + Gemma4-26B (15 GB) + other apps — easily >24 GB.
  • pykeen Adam state not checkpointed — every restart resets optimizer second-moment estimates. Every crash resets grokking progress. One-line fix.
  • MLX × PyKEEN contention — flock only serializes MLX×MLX. Consider extending the flock wrapper: "skip this run if pykeen CPU >100%".
  • Not all plists cleaned up — litestream, backup, log-rotate, morning-briefing, weekly-report still have default Nice/ProcessType (STABILITY-PLAN Group C2 deferred).
