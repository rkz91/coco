  <action>
Create the directory `~/.coco/hooks/` if it does not exist. Then create `~/.coco/hooks/log-event.js` following the exact pattern from `~/.claude/hooks/gsd-context-monitor.js`.

The hook script must:

1. **Read stdin with 3-second timeout** (same pattern as gsd-context-monitor.js):
   ```
   let input = '';
   const stdinTimeout = setTimeout(() => process.exit(0), 3000);
   process.stdin.setEncoding('utf8');
   process.stdin.on('data', chunk => input += chunk);
   process.stdin.on('end', () => { ... });
   ```

2. **Parse the hook event type** from stdin JSON. The hook handles BOTH SessionStart and PostToolUse:
   - If `data.tool_name` exists → PostToolUse event → log as type `tool_use`
   - If `data.tool_name` is absent → SessionStart event → log as type `session_start`

3. **Session ID: use data.session_id as PRIMARY** (not PPID):
   - `const sessionId = data.session_id || readSessionIdFile();`
   - data.session_id is the authoritative Claude Code session ID present in all hook stdin
   - PPID file is BACKUP only — read from /tmp/coco-session-$PPID.id if data.session_id is empty
   - On SessionStart, ALSO write session ID to /tmp/coco-session-$PPID.id as backup for edge cases

4. **Detect skill invocations** (approximation for EVNT-02):
   - If PostToolUse and `data.tool_name === 'Task'` or tool_input contains a slash command pattern (e.g., `/team`, `/gsd:`, `/pmstudio`, `/email`), set type to `skill_invoked` instead of `tool_use`
   - This approximates EVNT-02's skill_invoked/skill_complete requirement using available PostToolUse data
   - If tool_response exists and tool was a skill invocation, also log a `skill_complete` event with status

5. **Build event object** with this schema:
   ```
   {
     ts: Date.now(),
     type: 'session_start' | 'tool_use' | 'skill_invoked',
     session: data.session_id || fallback,
     tool: data.tool_name || null,
     input: sanitizeToolInput(data.tool_input),
     cwd: data.cwd,
     status: data.tool_response?.success !== false ? 'ok' : 'error'
   }
   ```

6. **Sanitize tool input**: Redact `content`, `new_string`, `old_string` fields (file contents) — replace with `'[redacted]'`. Keep all other fields (file_path, command, pattern, etc.).

7. **Ensure ~/.coco/ directory exists**: `fs.mkdirSync(dir, { recursive: true })` before writing.

8. **Rotate if events.jsonl > 50MB**: Rename to events.jsonl.bak, start fresh. This is defensive — real pruning is Phase 2.

9. **Append event**: `fs.appendFileSync(EVENTS_FILE, JSON.stringify(event) + '\n')`.

10. **Silent failure**: Wrap everything in try/catch, `process.exit(0)` on any error. Never block Claude Code tool execution.
