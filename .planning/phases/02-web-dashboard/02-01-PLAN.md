  <action>
  **Step 1: Initialize package.json in ~/.coco/**

  Create ~/.coco/package.json with:
  - name: "coco-dashboard"
  - type: "commonjs" (CJS — gray-matter is CJS, avoids ESM interop issues)
  - dependencies: express, gray-matter, open
  - Do NOT include @anthropic-ai/claude-agent-sdk (user has no API key; claude -p is primary per research finding)

  Run `cd ~/.coco && npm install` to install dependencies.

  **Step 2: Create ~/.coco/server.js with static serving + SSE events endpoint (~100 lines initially)**

  Express server with these initial endpoints:

  1. **Static serving:** `express.static('public')` for ~/.coco/public/
  2. **GET /api/events (SSE):** Tail ~/.coco/events.jsonl using fs.watchFile pattern from research:
     - Set SSE headers (Content-Type: text/event-stream, Cache-Control: no-cache, Connection: keep-alive)
     - Call res.flushHeaders()
     - Read all existing lines, send each as `data: {line}\n\n`
     - Track byteOffset, use fs.watchFile({ interval: 500 }) to detect new lines
     - Handle file rotation: if stat.size < byteOffset, reset to 0
     - Send heartbeat every 15s: `res.write(': heartbeat\n\n')`
     - On req close: fs.unwatchFile + clearInterval heartbeat
     - Handle missing file gracefully (create empty if not exists)

  **Middleware:** express.json() for POST body parsing. CORS not needed (same origin).

  **Server startup:**
  - Listen on PORT (env) or 3000
  - Log: "CoCo dashboard running at http://localhost:{PORT}"
  - Auto-open browser using `open` package (catch and ignore errors)
  - Log chat backend mode: "Chat backend: claude -p spawn (OAuth)"
  - **IMPORTANT:** Export the app/server for programmatic use: `module.exports = app;` — but only call listen() when run directly (`if (require.main === module) { ... }`)

  Create ~/.coco/public/ directory if it does not exist.
  </action>

<task type="auto">
  <name>Task 2: Add /api/chat spawn, /api/skills loader, /api/sessions grouper to server.js</name>
  <files>~/.coco/server.js</files>
  <action>
  Add remaining API endpoints to ~/.coco/server.js:

  1. **POST /api/chat (SSE response):** Spawn claude -p for chat:
     - Parse req.body.message (use express.json() middleware)
     - Set SSE headers + flushHeaders
     - Spawn: `spawn('claude', ['-p', message, '--output-format', 'stream-json'], { stdio: ['ignore', 'pipe', 'pipe'], env: process.env })`
     - Read stdout line-by-line, parse JSON, forward as SSE `data: {json}\n\n`
     - Buffer incomplete lines (split on \n, keep last if no trailing newline)
     - On proc close: send `data: {"type":"done"}\n\n` then res.end()
     - On req close: proc.kill() to cancel
     - Handle spawn errors gracefully (send error SSE event)
  2. **GET /api/skills:** Return cached skill catalog as JSON:
     - At server startup, call loadSkills() which reads:
       - ~/.claude/commands/**/*.md (recursive) — parse frontmatter with gray-matter
       - ~/.claude/skills/*/SKILL.md — parse frontmatter
     - For each skill: { id, name, description, family, source }
     - Family detection: gsd, pmstudio, email, team, superpowers, standalone (use detectFamily from research pattern)
     - Cache result in memory, serve as JSON array
  3. **GET /api/sessions:** Return session list as JSON:
     - Read events.jsonl, group by session ID
     - For each session: { id, status (active if last event < 60s ago), skill, duration, firstTs, lastTs, eventCount }
     - Filter to last 24h only
     - Sort by lastTs descending

  server.js should be ~250-300 lines total after this task.
  </action>
