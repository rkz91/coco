1. Autonomous Layer — Cron Jobs (launchd)

These run without you. The backbone of CoCo's autonomy.

JobScriptScheduleWhat It DoesVerdict
Master Cron master_cron.py 1:00 AM + 2:00 PM 13-phase Knowledge Engine across 21 projects: brain rescan → entity extraction → article generation → cross-project linking → graph build. 6hr timeout. CORE
Think Pass think.py Every 15 min Reads hub.db, classifies new items, applies attention rules (brain.json), auto-handles noise/FYI, queues by priority. Writes queue.json atomically. CORE
Email Watcher email_watcher.py Every 10 min Scans Outlook inbox for new emails. Extracts sender, subject, body, attachments. Routes to hub.db via Knowledge Hub ingest. CORE
MemPalace mempalace_service.py Always-on ChromaDB-backed semantic search service. Keeps vector embeddings warm. Listens on Unix socket. Used by search.py for semantic queries. Cold start ~8s without it. CORE
Morning Briefing morning_briefing.py 6:00 AM daily Generates JSON briefing: what happened in last 12h, new emails, decisions, action items. Saved to last-morning-briefing.json. Powers HomePage BriefingCard. CORE
Meeting Prep meeting_prep.py Every 15 min Checks calendar for upcoming meetings. Builds context from brain (attendees, project history, open items). Critical for the voice memo workflow. CORE
PyKEEN Train pykeen_bridge.py On-demand Trains knowledge graph embeddings using PyKEEN (TransE/RotatE). Top-50 predictions. This IS the GROKing — learning hidden relationships from the graph. CORE
Weekly Report weekly_report.py Weekly Auto-generates weekly status report from brain data. KEEP
Litestream litestream replicate Always-on Continuous SQLite replication to S3. Disaster recovery for all DBs. KEEP
Backup backup.sh Weekly Full backup script. KEEP
Log Rotate log-rotate.sh Weekly Rotates log files to prevent disk fill. KEEP
Verdict: All 11 cron jobs serve real purposes. 7 are CORE (the autonomous intelligence backbone). 4 are KEEP (infrastructure). None should be cut. The key insight: these run sequentially via master_cron.py — the master cron is the orchestrator that prevents parallel DB access conflicts.

What Needs RETHINKING (not cutting)

FeatureProblemProposed Fix
InboxPage (8 tabs) Cognitive overload. User freezes choosing a tab. Collapse to 3: Urgent / Drafts / Everything. Move Suggestions, Health, Auto-handled into smart filters within "Everything".
ProjectDetailPage (10 tabs) Kitchen sink. 2060 LOC monolith. Overview page + slide-over panels for each section. Max 4 visible tabs.
Insight Engine Generates insights but no UI surfaces them prominently. Cross-references are powerful but invisible. Feed insights into the Morning Briefing and HomePage cards instead of a separate page.
Podcast/Kokoro TTS Cool tech but script-only, no audio. Kokoro is 330MB download. Keep as opt-in "audio briefing" mode. Don't load by default.
Attention Tracker Tracks page views but doesn't influence routing decisions. Feed into action_router to prioritize agent work for neglected projects.
JarvisPage Overlaps with ChatPage. Both talk to Claude. Merge into Chat. Jarvis becomes a "mode" of Chat, not a separate page.
SelfImprove + ReplayPage Studio-only features. Niche audience. Keep behind feature gate. Useful for power users. Not part of daily journey.
Key finding: CoCo has zero bloat to cut. Everything serves a purpose. The problem isn't feature count — it's that the autonomous backend is extremely powerful (11 cron jobs, 70 scripts, 5 databases) but the frontend doesn't surface this work well. The V2 journey should be: CoCo does the work autonomously → the platform presents the results clearly → you review and approve in minutes.
