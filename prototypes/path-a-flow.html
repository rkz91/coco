CoCo Platform / UX Strategy

Path A -- Patch

Keep all 17 pages. Fix the 3 biggest pain points. Ship fast, learn faster.

1

Design Philosophy

Path A is the conservative, low-risk approach. Rather than rearchitecting the entire navigation, it patches the three most impactful friction points that users hit on every single session. The goal: reduce average task completion from 6 steps to 2-3 steps without retraining anyone.

⚔️

Inline Actions

No more page-hopping just to click a button. Approve, complete, and dismiss directly from the Home dashboard.

📥

Simplified Inbox

3 tabs instead of 8. Consolidate noise into Actionable, Informational, and Archived streams.

📋

Structured Agent Output

Summary first, raw log on expand. Stop forcing users to parse streaming JSON to understand what happened.

Pros & Cons

Pros

  • Low risk -- incremental changes, easy to roll back individually
  • Preserves familiarity -- zero retraining needed for existing users
  • No data migration -- all existing data stores untouched
  • Fast to ship -- each fix is an independent PR, can deploy one at a time
  • Validates assumptions -- if inline actions don't help, we learn cheaply

Cons

  • Still 17 pages -- structural complexity is reduced but not eliminated
  • Cognitive overhead remains -- "where is this?" question persists
  • Navigation debt untouched -- sidebar still has 17 links
  • Diminishing returns -- further patches will yield smaller gains
  • Doesn't address discoverability of less-used features

Effort Estimate

Sizing assumes a single developer familiar with the codebase. Each change is independently shippable.

Change Size Reasoning
Inline action cards on Home Medium New component + wiring approve/complete/dismiss handlers to existing API endpoints. Needs optimistic UI updates via TanStack Query.
Simplified inbox (3 tabs) Small Purely frontend filter logic. Collapse 8 categories into 3 buckets. No backend changes.
Structured agent output Medium Requires summary extraction from agent stream (backend) + collapsible card component (frontend). May need an LLM summarization pass on completion.
Decision queue inline on Home Small Subset of inline action cards. Reuses the same component pattern with approve/reject variants.
Total Medium Roughly 1-2 weeks of focused work. No backend schema changes. No migrations.
