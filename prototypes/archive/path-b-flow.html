CoCo Platform / UX Strategy

Path B -- Collapse

17 pages become 4. Everything else is a slide-over panel. Navigate by acting, not by page-hopping.

"4 pages, infinite depth."

1

Design Philosophy

Home is the command center. Everything else is a slide-over panel. The user never leaves Home for routine work. They act on items inline, or click to open a panel that slides in from the right. Escape closes it. The stream is always there underneath.

Command Center Model

Home shows the action stream: decisions, agent updates, todos, costs. Every item is actionable inline. No dead-end display pages.

Slide-Over Panels

Need more detail? Click an item and a panel slides in from the right. It stacks, so you can drill deeper. Escape pops the stack.

Navigate by Acting

You don't navigate to "the Costs page." You see a cost alert in the stream and click it. Context comes to you, not the other way around.

Keyboard-First

Escape = close panel. Tab = next item. Enter = open detail. Arrow keys = scroll. Power users never touch the mouse.

Page Reduction: 17 to 4

The current app has 17 separate pages. Path B collapses them into 4 real pages plus a stack of slide-over panels. Most "pages" become either absorbed into Home's stream, converted to panels, or killed entirely.

17 Pages to 4 Pages + Panels

Core User Flow

The primary loop: open CoCo, see the action stream, act inline or open a slide-over, close panel, back to stream. The user never leaves Home for routine work.

Primary User Journey

When You Leave Home

Only three reasons to navigate away from Home:

Agents Page

Monitor long-running agents. See all active processes, their progress, and output summaries. Agent detail opens as a slide-over.

Tree Page

Reorganize your portfolio. Move projects between teams, adjust hierarchy, search knowledge. Project/knowledge detail opens as slide-over.

Slide-Over Panel Stack

Panels stack from right to left. Each panel pushes the previous one slightly left and dims it. Pressing Escape pops the top panel. The stream is always visible underneath as a faded background.

Panel Stacking Behavior

Stack Rules

Max Depth: 3

At most 3 panels can stack. Opening a 4th replaces the bottom panel. This prevents "panel hell."

Escape = Pop

Escape always closes the topmost panel. Clicking outside the panel stack also closes the top. Muscle memory from modal UIs.

Pros & Cons

Pros

  • Massive complexity reduction -- 17 pages to 4 pages
  • One-place workflow -- 90% of work happens without leaving Home
  • Keyboard-friendly -- Escape to go back, Tab/Enter to navigate
  • Modal muscle memory -- slide-overs feel familiar from Notion, Linear, Slack
  • Sidebar shrinks from 17 links to 4 -- instantly less overwhelming
  • Context preserved -- the stream is always visible behind panels
  • Easier to build new features -- just add a panel type, no new route

Cons

  • Slide-over panels can feel cramped for complex content
  • Loses the "full page" feel for deep work (e.g., long agent output)
  • Chat as a panel may feel limited vs. a dedicated page
  • Higher upfront effort -- requires rearchitecting navigation + layout
  • Panel state management adds complexity (stack, animations, focus trap)
  • Risk of "panel fatigue" if too many things are crammed into slide-overs
  • Deep links harder -- URL routing for panel state is non-trivial
