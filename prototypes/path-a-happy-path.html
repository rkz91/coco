Overview

The Happy Path Flow

Every step leads forward. Every action gives feedback. No confusion, no dead ends.

Breakdown

Step-by-Step Detail

Each step shows what the user sees, what they do, and the system response.

 Home with briefing cards Open browser tab Action cards load instantly 1s

 "3 urgent, 5 drafts, 1 running" Glance at attention strip Knows what needs attention 2s

Scope

What's NOT in the Happy Path

Intentionally excluded. Valid features, but not the daily workflow.

×Settings configuration
×Tree / portfolio management
×Cost analysis deep dive
×Knowledge base browsing
×Agent org chart
×Self-improve cycles
×Batch operations
"These are all valid features, but they're not the daily workflow. Fix the happy path first, everything else can wait."
Acceptance

Success Criteria

After implementing the Path A happy path, every one of these must be true.

Home loads in < 1s
Any action card can be acted on in 1 click
Agent spawn takes < 5s from Orb click to agent running
Agent completion shows structured summary (not raw log)
Chat can be opened with pre-loaded context in 1 click
Toast confirms every action within 200ms
Roadmap

Implementation Priority

Build in this order. Each step unlocks the next piece of the happy path.

1
Inline action buttons on Home cards
Approve, complete, dismiss -- without navigating away
2
Structured agent summary cards
Replace raw logs with readable, scannable summaries
3
Quick spawn from Orb
Type a task, hit Spawn, agent starts immediately
4
"Chat About" with context pre-loading
One click opens chat with the relevant agent output already loaded
5
Simplified inbox (3 tabs)
Urgent / Drafts / Running -- nothing else on the home screen
6
Running cost on agent cards
Live cost counter so you always know what you're spending
7
Everything else
Settings, tree view, knowledge base, org chart, batch ops
