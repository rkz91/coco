Foundation — Cost Architecture

$4,033 IN 15 DAYS IS NOT SUSTAINABLE

The problem: Building and running CoCo has cost $4,033.73 in the first 15 days of April 2026. At this rate, that's $8K/month. The Knowledge Engine generates ~1,700 articles across 21 projects, processes 40-80 emails/day, trains graph embeddings, and runs a 10-agent self-improvement squad — all hitting the Anthropic API. Most of this work does NOT need Claude Opus or Sonnet.

Foundation — Memory Infrastructure

THE BRAIN BEHIND EVERYTHING

CoCo's memory is not a single database — it's a 5-layer system where each layer serves a different access pattern.

How Data Flows Through Memory

Sequential execution is critical. The Master Cron (master_cron.py) runs phases 1-6 per project sequentially, then phases 7-10 globally. This prevents SQLite write conflicts. The pipeline takes ~2-4 hours for all 21 projects. It runs at 1:00 AM and 2:00 PM.

Foundation — Cron Orchestration

10 LAUNCHD JOBS — THE AUTONOMOUS BACKBONE

These are the actual launchd plists running on the Mac. This is what makes CoCo autonomous.

No parallelism rule: The cron jobs are designed to run sequentially. Email Watcher writes to hub.db → Think Pass reads hub.db and writes queue.json → Master Cron reads hub.db and writes knowledge.db → Morning Briefing reads everything. Running them in parallel would cause SQLite lock conflicts. The StartInterval staggering and master_cron.py orchestrator enforce this.

Phase 1 — Email Triage

FIRST THING IN THE MORNING

The pain: Outlook opens. 40-80 emails. A wall of threads — vendor updates, meeting invites, project replies, FYI forwards, compliance alerts, approvals. No way to know what actually matters without reading each one. This is 30-60 minutes of dead time every morning.

What CoCo Does Autonomously (Before You Open It)

What Rijul Sees When He Opens CoCo

Autonomy principle: CoCo doesn't ask permission to read and analyze. It asks permission to send and to write to shared docs. Everything else happens in the background. The human gate is: approve the reply, approve the doc update, or override.

The 10-Agent Team Per Project

Each project folder has a dedicated squad. When an email triggers document updates, CoCo doesn't use one monolithic agent — it dispatches from the team:

AgentRole in Email FlowTrigger
Researcher Loads current project state, recent brain updates, open threads Always — first agent to run on any email batch
Analyst Extracts new facts, dates, requirements, action items from emails When email contains new information
Writer Updates PRD, BRD, meeting notes, decks with new info When Analyst flags document-worthy content
Reviewer Cross-checks updated docs against source email for accuracy After Writer finishes any doc update
Communicator Drafts reply emails matching Rijul's tone and project context When email needs a response
Archivist Updates Brain DB: entities, decisions, events, relationships After all processing — captures what happened

The remaining agents (Planner, Developer, Tester, Presenter) activate in later phases — not during email triage.

Phase 1b — Meetings & Voice Memos

DURING AND AFTER MEETINGS

The pain: You leave a 45-minute meeting with a voice memo, action items in your head, and a Zoom recording link in your email. The meeting notes don't get written for hours (or ever). Documents that should be updated with what was discussed stay stale. Follow-up replies never get sent because you forgot who said what.

How CoCo Handles Meetings

When a voice memo or Zoom summary arrives, CoCo builds calendar context on-demand — no separate cron job needed. It checks your calendar, matches the meeting, loads project context from the brain, and processes everything in one shot.

Voice Memo Flow (Existing: voice_memo_watcher.py)

Key insight: Apple Voice Memos already provides transcriptions natively. CoCo uses AppleScript to extract the transcription directly — no API cost, no Deepgram, no Whisper needed. Whisper.cpp is kept as a fallback for older memos or if Apple transcription isn't available. This is a $0/month operation.

Zoom Summary Flow

When Zoom sends a meeting summary email:

StepWhat Happens
1 Email Watcher detects Zoom summary email (sender: no-reply@zoom.us)
2 CoCo shows in inbox: "Zoom summary received for [Meeting Name]. Want me to process it?"
3 You tap Accept — one click. (Or provide your own summary if Zoom's isn't great.)
4 Agent processes: calendar context + Zoom summary → meeting notes + action items + doc updates
5 Same output pipeline as voice memos: notes in folder, todos created, brain updated, docs flagged

What Rijul Sees in CoCo

Cost for meeting processing: ~$0. Transcription is free (Apple native). Notes generation runs on local Gemma. Only the follow-up email drafts hit Claude API (~$0.05 each). A 5-meeting day costs less than $0.25 total.
